================================================================================
TROUBLESHOOTING
================================================================================

COMMON ISSUES AND SOLUTIONS:

1. "Repository already exists" error:
   - Check if you already have a repository with this name
   - Try a different name like "TrackTx-NF" or "TrackTx-Pipeline"

2. "Authentication failed" error:
   - Use GitHub CLI: `gh auth login`
   - Or use personal access token instead of password
   - Or use SSH keys: `git remote set-url origin git@github.com:USERNAME/REPO.git`

3. "Large files" error:
   - Check .gitignore is working: `git status`
   - Remove large files: `git rm --cached largefile.bam`
   - Use Git LFS for large files if needed

4. "Permission denied" error:
   - Make sure you own the repository
   - Check repository is public if you want public access
   - Verify your GitHub username is correct

5. GitHub Actions failing:
   - Check the Actions tab for error details
   - Verify Docker is available in the workflow
   - Test locally first: `./run_pipeline.sh --dry-run`

6. Pages not building:
   - Check Settings ‚Üí Pages for error messages
   - Ensure README.md is in the root directory
   - Wait a few minutes for deployment
